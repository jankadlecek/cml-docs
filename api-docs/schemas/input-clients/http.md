# HTTP Input Client

> **Context for AI:** The HTTP input client fetches data from REST APIs.
> It is the most commonly used input client in CML orchestrations.
> Supports all HTTP methods, authentication, pagination, retries,
> custom headers, and response parsing (JSON/CSV/XML).
>
> **Key config fields:**
> - `url`: Target API URL
> - `method`: HTTP method (GET, POST, etc.)
> - `auth`: Reference to an auth configuration
> - `pagination`: Automatic pagination handling
> - `responseHandler`: How to parse the response
> - `retryable`: Retry configuration for failed requests
>
> **See also:** [Shared input types](_types.md), [Auth types](../auth/auth-types.md)
